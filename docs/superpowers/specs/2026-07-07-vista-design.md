# VISTA Design — views overhaul, Claude-skills showcase, polish, features

Date: 2026-07-07 · Produced by a 6-agent design workflow (2 competing view architects, showcase, polish auditor, feature designer, xhigh judge). Full JSON in session scratchpad (vista-design.json).

## User brief (verbatim)
> "multiple more graph views… absolutley stunning no excuses… revamp the hud… more space in between folders and notes so we can see everything more clearly and actually look through our notes… keep adding new features… completely polished app"

## Judge ranking
- **9.2** [clarity] Stacks (Folder Shelf Hall) — Best direct answer to 'actually look through our notes': zero-occlusion 2D grid bent into 3D, alphabetical rows, left-anchored labels via stock sprite.center. Verified the row math against label.js (2.7u text vs 4.6u rows) and its bloom-threshold caveat is real. Cheapest per-frame view (static, no sim). Completely distinct from all 5 existing lenses — nothing today is structured/readable. Minor risk: rest-camera 330u puts labels near ~9px; may need scale 0.06 or nearer start camera.
- **9** [clarity] Atlas (Cluster Archipelago) — Only proposal with testable spacing invariants (min 14u intra-island, GULF*0.9 between hulls) sized off the actual 30u label footprint. Pan-a-map browsing plus bioluminescent island discs hits spacing AND stunning. Reuses GOLDEN (already in GlobeView:11), _flyTo and click/dbl patterns verbatim from BrainView. Static layout = trivially 60fps. Distinct: no existing view is top-down cartographic or cluster-island based (Constellation is folder-ring).
- **7.8** [cinematic] ArchiveCityView (skyline by folder district) — The release's money screenshot — neon districts, window-dot emissive maps, fake reflections, volumetric beacons, all zero-dep and static (~320 simple meshes, integrated-GPU safe with the reflection clone as the first thing to drop if a shot-check misses 60fps). Height=degree adds a real analytic read without labels. Weaker on browse (note labels only under d=60, links off by default) and folder-grouping overlaps Solar/Constellation conceptually, but visually it duplicates nothing.
- **7** [clarity] Chronicle (Timeline River) — Genuinely new axis (time) no lens covers, lightest per-frame cost, and by-construction only ~25-35 labels in the fade window. Cut anyway: mtime is last-touch (dashboard.mjs says so explicitly, OneDrive can rewrite it), so the river would lie about creation-time for ~70 of 103 notes; plus the wheel-to-time remap is the release's riskiest interaction change. Resurrect once first-seen dates are persisted in the snapshot store.
- **6** [cinematic] FilamentView (cosmic web) — Beautiful and perf-safe, but its macro job (clusters far apart, filaments between) is exactly what BrainView delivers after the force-sim rescale, at half the novelty; technique (fibSphere) echoes Globe. The 'uncrowded neighborhood to browse' claim fails arithmetic: ~7.8u min chord in a 20-note group vs ~30u labels — labels collide the moment you fly in. Duplicative and doesn't fix the pain.
- **5.5** [cinematic] CortexView (neural cortex sheet) — The triple-layer membrane is the prettiest single object proposed, but S=5.0 Vogel gap guarantees orb non-overlap, not label non-overlap — 5u pitch under ~30u labels means 'default labels ON' produces exactly the cramped mush the user complained about (the claim they 'don't collide' is wrong by 6x). Metaphor also duplicates BrainView (it IS the neural view). Wrong view to spend the wow budget on.

## Cuts
- CortexView (cinematic) — CUT. Its 5u Vogel gap guarantees orbs don't touch but labels are ~30u wide; 'default labels ON, they don't collide' is wrong by ~6x, so it re-creates the cramped mush the user is escaping. Neural metaphor duplicates BrainView, which becomes the spacious neural view after slice 2. The membrane idea is worth stealing later as a BrainView floor, not a sixth lens.
- FilamentView (cinematic) — CUT. Post-rescale BrainView + Atlas already deliver 'clusters far apart with visible bridges'; in-group 7.8u spacing breaks the browse promise; fibSphere technique echoes Globe. Three cluster-macro lenses is two too many.
- Chronicle (clarity) — CUT from this release, best of the losers. mtime is last-touch (dashboard.mjs labels it honestly; OneDrive sync can rewrite it wholesale), so the time river would misrepresent ~70 of 103 notes; wheel-remap is the riskiest proposed interaction. Revisit as fast-follow once first-seen dates are persisted in the snapshots store — the layout math itself is sound.
- Orphan Triage feature — DEFER to fast-follow. Its engine (suggest.mjs + accept handler + toast-undo) ships entirely inside Synapse Suggestions; the modal is 170 lines that can land the week after without blocking anything.
- Vault Time Machine feature — CUT. Same mtime-honesty problem as Chronicle ('watch the brain grow' via last-edit dates shows edits, not growth), and the release already carries three features. Rides activeIds so it loses nothing by waiting.
- Saved Filter Chips feature — CUT. Least aligned with spacing/browse/brain brief; pure convenience. Later.
- Clarity's Solar orbitR formula — dropped in favor of Cinematic's floor(k/5) ring-pitch version (real concentric rings, pitch 3.4 > max planet diameter, cleaner eclipse-free motion); Clarity wins everything else in the Solar retrofit (ring 120, camera, labels, incident-only hover).
- Cinematic's force.mjs numbers — superseded by Clarity's more aggressive, internally consistent package (fade window and camera matched to the new radius); one reconciled set only.
- Polish items 11-25 — take the listed top 10 only this release; the rest queue behind the release pass.

## Risks
- Force-sim rescale (slice 2) changes BrainView's whole character: repulsion 900 + cutoff2 13000 may equilibrate slowly or oscillate with the 300-tick warmup; budget one constant-tuning iteration against before/after screenshots before touching other views.
- label.js scale compensation touches every makeLabel caller across 5+ files; one missed call site renders giant text. Grep-audit all callers in the same commit as the FONT change.
- Bloom threshold 0.06→0.3 exempts labels but may also mute dark-colored orbs whose emissive luminance falls below 0.3 — verify gem glow per view in the slice-1 screenshots and tune per-view strength (they already differ) rather than sharing one number.
- Stacks rest-camera legibility is marginal on paper (~9px glyphs at 330u with 0.055 scale); acceptance must be a screenshot with real vault titles, not the math — fallback is scale 0.06 + camera 280 which the column widths already tolerate.
- ArchiveCity transparency overdraw (26 beacon shafts + 103 reflection clones + bloom) is the only real integrated-GPU risk in the release; measure with ONYX_SHOT and cut reflections, then shafts, in that order.
- Synapse Suggestions is the only vault-writing feature: OneDrive file locks and sync races can corrupt or resurrect content. Keep the readNote-null abort, whole-file-restore undo, and never write from the dashboard panel without re-reading the file first.
- Suggestion precision on a betting vault is unproven (lexical rare-term overlap may pair unrelated notes via player names); the dismissed-store and per-note cap of 3 bound the annoyance, but if acceptance rate is low in practice, tighten score threshold before considering TF-IDF.
- Release scope is ~2500 lines across 3 new views + 8 retrofit fixes + 3 features + showcase + 10 polish items: the cut list is the schedule. If anything slips, ArchiveCity (slice 9) and Showcase (slice 13) are the drop candidates — Stacks/Atlas/spacing/polish are the brief's core.
- SpaceCanvas fully rebuilds on every view switch; three new views mean three new dispose paths (canvas textures, reflection clone, disc sprites). Leak-test by cycling all 8 lenses repeatedly and watching renderer.info.memory before release.
- detectClusters output is layout input for Atlas: cluster count/membership shifts as links are added (especially once Synapse Suggestions lands), so island layouts will reshuffle after accepting suggestions — expected behavior, but note it in the changelog so it doesn't read as a bug.

## Winning view specs (full)
### Atlas (Cluster Archipelago) [clarity]

**Concept:** Top-down map of 17 cluster 'islands' separated by wide empty gulfs. Each island is a flat sunflower disc of notes sized so every label has its own footprint — you pan across the map and read every title at rest, like Google Maps for the vault. Orphans form a faint outer reef ring.

**Layout:** Two-level golden-spiral packing, all deterministic from detectClusters(ids, links). CONSTANTS: FOOT=30 (world width of one label: 720px canvas x 0.042 scale ~ 30u), GOLDEN=PI*(3-sqrt(5)) (already in GlobeView.js line 11). LEVEL 1 (notes within cluster ci, members sorted by degree desc so hubs sit center): member j at r_j = 0.62*FOOT*sqrt(j), theta_j = j*GOLDEN, y = min(deg,12)*0.8 (hubs rise slightly). Island radius R_ci = 0.62*FOOT*sqrt(n_ci) + FOOT. For the real vault (mean 6 notes/cluster, max ~20): R ranges ~48 to ~113. LEVEL 2 (island centers, clusters sorted by size desc): center_i = (S*sqrt(i+0.7)*cos(i*GOLDEN), 0, S*sqrt(i+0.7)*sin(i*GOLDEN)) with S = max_ci(R_ci) + GULF, GULF=45; after placement run 20 passes of pairwise center push-apart: if dist(a,b) < R_a+R_b+GULF, move each half the deficit along the separation axis (17 centers = 136 pairs, trivial). Orphans (clusterOf == -1) on a ring at radius = max island extent + 100, spaced FOOT*1.2 apart by arc length. Acceptance math: min intra-island note spacing = 0.62*FOOT*(sqrt(j+1)-sqrt(j)) neighbors on adjacent spiral turns ~ 0.62*FOOT ~ 18.6u center-to-center laterally, with labels offset above orbs and camera top-down so lateral spacing is what matters; assert in test that no two note positions in a cluster are closer than 14u and no two island hulls closer than GULF*0.9.

**Interactions:** Default camera high oblique (0, 420, 300) looking at origin, OrbitControls with enablePan=true and maxPolarAngle=PI/2.4 so it stays map-like. Click island disc = _flyTo framing that cluster top-down (reuse BrainView._flyTo pattern lines 219-226). Single-click note = pin + hover card via existing onHover contract; double-click = onSelect (open note), matching BrainView lines 195-217. setActive dims non-matching notes AND shrinks their islands' disc glow. Inter-cluster links render as elevated QuadraticBezierCurve3 arcs (midpoint y = 18 + 0.08*dist) so gulf floors stay visually empty; intra-cluster links flat at y=0.3, opacity 0.12.

**Wow:** Each island floats on an additive-blended radial-gradient disc sprite tinted CLUSTER_PALETTE[ci % 12] at opacity 0.10 — reads as bioluminescent shallows around each archipelago. Inter-cluster arcs carry LinkPulses (scenery.js class, count 60) so ideas visibly ferry between islands. A big cluster-name label per island (title of highest-degree member, makeLabel scale 0.10, color = palette hex) floats at y=16 like a map place-name. makeNebula('#0d2038','#161042') gives a deep-ocean floor; fog FogExp2 0.0009 so the far islands haze out like real geography.

**Implementation:** New pure function archipelagoLayout(ids, links, clusterOf, degOf) -> Map<id,{x,y,z}> + island metadata in NEW src/renderer/lib/layouts.mjs with node --test asserting the two spacing invariants (zero deps, pure math). View class copies the GraphView.js skeleton (static layout, no per-frame sim — cheapest of all views): makeOrb per note, one LineSegments for flat intra links, one THREE.Line per inter-cluster arc built from curve.getPoints(12) merged into a single LineSegments buffer, LinkPulses on the arc buffer, makeLabel per note with the BrainView distance-fade loop (fade 200->420 since map camera is far). Discs = 17 sprites (softDot() texture from scenery.js, tinted). 103 orbs + 103 sprites + 2 line buffers + 17 discs: far under the existing BrainView draw load, easy 60fps. Register in SpaceCanvas.jsx VIEWS map + ViewSwitcher entry.

### Stacks (Folder Shelf Hall) [clarity]

**Concept:** A library hall: 13 folders become 13 glass columns in a shallow arc, every note a labeled row you can literally read top to bottom without touching the mouse. This is the 'actually look through our notes' view — zero occlusion by construction, because layout is a 2D grid bent into 3D.

**Layout:** CONSTANTS: COL_W=48 (> max label width 30u + gutter), ROW_H=4.6 (label world height = 60px x 0.045 = 2.7u, so 1.9u clear gap), MAX_ROWS=18, TOP_Y=42. Folder i of 13 (ordered as graph.folders) at xBase_i = (i - 6) * COL_W = -288..+288; arc depth z_i = 0.0011 * xBase_i^2 (ends recede ~91u, center column nearest camera). Notes within folder sorted alphabetically (daily logs auto-sort chronologically since they are date-named). Note k: lane = floor(k/MAX_ROWS), y = TOP_Y - (k % MAX_ROWS)*ROW_H, x = xBase_i + lane*20 - (numLanes-1)*10 (lanes centered). '06 - Daily Logs' with ~30 notes = 2 lanes of 18+12 — total column width 40u, still inside COL_W=48. Tallest column height = 18*4.6 = 82.8u. Orb at (x,y,z_i), radius 0.9 fixed (uniform = calmer); label sprite with sprite.center.set(0, 0.5) anchored 1.6u right of orb so text never overlaps its orb and rows left-align like a list. Folder header: makeLabel(f.name, f.color, 0.10) at (xBase_i, TOP_Y + 8, z_i). Acceptance math: default camera (0, 8, 330), FOV 50 -> center column at ~330u; use label scale 0.055 (font legible to ~380u given 40px texture) and fade window 260->520 so every visible row is >= 0.6 opacity at rest.

**Interactions:** OrbitControls with enableRotate=false-ish (minPolarAngle=maxPolarAngle=PI/2.15 lock, azimuth clamp +/-0.35) — pan and dolly only, so it always reads as a hall, never becomes spaghetti from behind. Hover note = draw its links as golden elevated arcs leaping across columns (build incident-only Float32 buffer exactly like BrainView.hlLines lines 120-126, but through QuadraticBezierCurve3 with mid z pulled toward camera +25 so arcs fly IN FRONT of columns); ambient state shows NO links at all — the hall stays clean. Click = pin + hover card, double-click = onSelect open (BrainView click/dbl pattern). setActive dims rows to 0.12 like every other view — combined with the search filter this becomes a spatial finder.

**Wow:** Columns are thin glass slabs: BoxGeometry(COL_W-10, laneRows*ROW_H+6, 0.6), MeshStandardMaterial transparent opacity 0.05 + an additive LineSegments edge frame in the folder color — Tron shelving. A floor: THREE.GridHelper(1200, 60, 0x24304f, 0x141b30) at y=-6 receding into FogExp2(0x05060a, 0.0016) sells the hall depth. Hover-arc gets a single LinkPulses instance (count 8) running along it — a spark jumps from note to its references. Folder headers get a rule-tick underline (11-segment LineSegments) echoing the .rule-ticks chrome kit from index.css so 3D and 2D UI rhyme.

**Implementation:** shelfLayout(folders, notes) -> Map<id,{x,y,z}> + column metadata in lib/layouts.mjs (pure, tested: assert row gap == ROW_H, no two positions equal, lanes cap at MAX_ROWS). View class from GraphView.js skeleton. Reuse makeOrb (pass note.type for gem shape), makeLabel, makeStarfield(600) sparse, makeNebula. sprite.center is a stock THREE.Sprite property — no new code needed for left-anchoring. Hover arcs rebuilt only on hoverId change (not per frame; columns are static). 103 orbs + 116 sprites + 13 slabs + 13 edge frames + grid = trivial draw list; the only per-frame work is animateOrbs + label fade. Skip bloom threshold below 0.2 here or the white text smears — this is the reading view.

### ArchiveCityView (skyline by folder district) [cinematic]

**Concept:** The vault as a neon archive-city at night seen from a drone: 13 folder districts on a dark reflective ground plane, each note a glowing monolith tower whose height is its link degree. Districts have wide boulevards between them and towers sit on a generous grid, so you can literally read the skyline: tall towers = hub notes, district color = folder. This is the most instantly-legible 'where is everything' view and the best screenshot.

**Layout:** In lib/layouts.mjs. (1) Districts: folder i of F placed on a golden-angle spiral, Rd = 42 * sqrt(i + 0.6), thetaD = i * 2.399963; district center = (Rd*cos, 0, Rd*sin). For 13 folders outermost R≈155, and 42·sqrt pitch keeps adjacent district centers ≥ ~38 apart. (2) Towers inside a district on a centered square grid with wide streets: cols = ceil(sqrt(m)), PLOT = 7.0; tower k at gx = (k % cols - (cols-1)/2) * PLOT, gz = (floor(k/cols) - (rows-1)/2) * PLOT, rotated by thetaD + PI/2 so every district's grid faces the city center (streets radiate — cinematic and orienting). District footprint = cols*PLOT; largest folder (~20 notes → 5x4 grid) spans 35x28, fitting inside the 38-unit pitch with ≥6-unit boulevards. Deterministic note order: sort by degree desc so the tallest tower is at grid center-ish (fill order spiral: sort grid cells by |gx|+|gz| ascending, assign biggest-degree first) — each district reads as a peak. (3) Tower height Hk = 4 + 2.4*sqrt(deg), capped 20; tower base 2.2 x 2.2. (4) Ground plane y=0; camera starts (0, 95, 190), controls.maxPolarAngle = 1.45 (never below ground).

**Interactions:** Hover: raycast towers (same _pick), lift the tower group +0.8 with a lerp and raise emissiveIntensity 0.9→1.8, cursor pointer, stream onHover card. Click = onSelect (towers are unambiguous targets — no pin dance needed). Click a district beacon = _flyTo(district center + (0, 26, 40)). District labels ALWAYS visible: makeLabel(folder name, folder color, 0.085) floating at (center.x, 26, center.z) — 13 labels never collide at 38+ unit pitch. Note labels appear only under distance 60 (fade like BrainView) to keep the skyline clean; setLabels(true) forces all on. setActive dims non-matching towers to opacity 0.15 and kills their reflection clone.

**Wow:** Towers: shared BoxGeometry(1,1,1) scaled (2.2, Hk, 2.2), MeshStandardMaterial color = folder color darkened 0.35, emissive = folder color, emissiveIntensity 0.9, plus a shared canvas emissiveMap of tiny window dots (64x256 canvas, 3px dots on black, THREE.NearestFilter, repeat wrapping) — bloom turns the windows into lit offices. Each tower capped by a 0.5-unit additive 'crown' sprite (softDot, folder color) that bloom flares. Fake reflections: one clone group of all towers with scale.y = -1, material transparent opacity 0.14, emissiveIntensity 0.35, depthWrite false — instant wet-street mirror, +103 cheap draw calls. Ground: CircleGeometry(240) MeshStandardMaterial 0x05060c, roughness 0.35, metalness 0.85 so the rim light streaks across it, plus a polar-grid canvas texture (rings every 21 units, radial spokes at each district angle, 1px #1c2a55 lines) as its emissiveMap at 0.25 — the city sits on a holo-deck. Volumetric beacons: per district a vertical light shaft = PlaneGeometry(3, 34) x2 crossed at 90°, MeshBasicMaterial with a vertical-gradient canvas (folder color → transparent), additive, depthWrite false, always-face-camera not needed since crossed pair reads volumetric from all angles; slow opacity breathe 0.10±0.04. FogExp2 0x04050a 0.0035 swallows the far districts into colored haze — strong depth-of-field feel. Keep makeStarfield(1200) + makeNebula('#101736','#081226') for the sky.

**Implementation:** New file src/renderer/views/ArchiveCityView.js registered as 'city'. Reuse addLights, makeStarfield, makeNebula, makeLabel, softDot; NOT makeOrb (boxes read as architecture; gem orbs would fight the metaphor). New layouts.mjs exports: districtGrid(m, plot) → array of {gx,gz} sorted center-out, with a node --test asserting no two cells closer than plot and count == m. Window texture + gradient shaft texture built once, module-cached like softDot. Perf: ~210 static meshes + 26 shaft planes, zero per-frame buffer writes; animation is only crown pulse (scale) and shaft opacity — integrated-GPU safe. Links deliberately OFF by default in this view (setLinksMode toggles a straight LineSegments buffer between tower tops, opacity 0.1 additive) — the skyline is about places, not edges; keeps it uncluttered. Dispose: track clone group + textures for disposal.
